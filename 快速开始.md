# 快速开始指南

## 📁 目录结构规范

```
frontend-86-projects/
├── projects/                  # 所有项目（按序号命名）
│   ├── 01-parallax-scrolling/
│   │   ├── index.html
│   │   ├── style.css
│   │   ├── script.js
│   │   └── README.md          # 项目说明+学习笔记
│   └── 02-fullscreen-video/
├── assets/                    # 公共资源
├── index.html                 # 导航页
└── README.md                  # 总介绍
```

**核心原则**：每个项目独立完整，可单独复制使用。

---

## 🚀 开始新项目

### Step 1：创建项目文件夹
```bash
mkdir projects/03-project-name
cd projects/03-project-name
```

### Step 2：编写项目文件
创建 `index.html`、`style.css`、`script.js`

### Step 3：本地测试
右键 `index.html` → 用 Live Server 打开

### Step 4：写项目说明
在项目内创建 `README.md`：
```markdown
# 项目名称

## 实现功能
- 功能1
- 功能2

## 技术要点
- 技术点1
- 技术点2

## 踩坑记录
- 问题描述及解决方案
```

### Step 5：提交代码
```bash
git add projects/03-project-name/
git commit -m "完成项目03：项目名称"
git push
```

## 📋 Git 工作流

### 提交规范
- **一个项目 = 一次提交**
- 提交信息：`完成项目xx：项目名称`
- Bug 修复单独提交：`修复项目xx中的xxx问题`

### 里程碑标签
每完成10个项目打一个标签：
```bash
git tag -a v1.0 -m "完成前10个项目"
git push origin --tags
```

## 🌐 GitHub Pages 部署

### 开启 Pages
仓库 Settings → Pages → Source: `main` branch

### 访问地址
```
https://你的用户名.github.io/frontend-86-projects/
```

### 更新导航页
每次完成新项目，在 `index.html` 中添加项目卡片

---

## 📌 注意事项

✅ **推荐做法**
- 每个项目独立，不跨文件夹引用
- 使用 CDN 或相对路径引用资源
- 及时写 README 记录学习心得
- 遇到问题记录在项目 README 中

❌ **避免事项**
- 不要把所有代码写在一个文件
- 不要依赖其他项目的文件
- 不要提交 `node_modules/`、`.DS_Store` 等临时文件
