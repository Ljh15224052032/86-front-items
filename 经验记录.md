## 一、语法错误类

### 1.1 HTML语法问题

#### 项目01：script标签语法错误

**现象**：JavaScript代码不执行

**原因**：
- ❌ `<script type="text/javascript" let bg=...` 语法错误
- ❌ 在JS中使用了HTML注释 `<!-- -->`

**解决方案**：
1. ✅ 正确的script标签写法
   ```html
   <!-- 引入外部JS文件 -->
   <script src="script.js"></script>

   <!-- 或内联代码 -->
   <script>
       let bg = document.getElementById('bg');
   </script>
   ```

2. ✅ 使用JS注释而非HTML注释
   ```javascript
   // 错误：<!-- 这是HTML注释 -->
   // 正确：// 这是JS单行注释
   /* 正确：这是JS多行注释 */
   ```

**知识点**：
- `<script>` 标签的 `type` 属性后不能直接跟代码
- JS中使用 `//` 或 `/* */` 注释，不能用 `<!-- -->`

#### 项目02-问题2：HTML结构与CSS选择器不匹配

**现象**：CSS样式不生效，内容层无法显示

**原因**：
- ❌ CSS使用 `.banner .content` 选择器（查找banner内部的content）
- ❌ HTML中 `.content` 和 `.banner` 是兄弟关系，不是父子关系

**错误代码**：
```html
<!-- HTML结构 -->
<div class="banner">
    <video>...</video>
</div>

<div class="content">  ← 在banner外面！
    <h1>...</h1>
</div>
```

```css
/* CSS选择器 */
.banner .content {  /* 这个选择器找不到元素 */
    z-index: 1;
}
```

**解决方案**：
1. ✅ **方案1：修改HTML结构**（推荐）
   ```html
   <div class="banner">
       <video>...</video>
       <div class="content">  ← 移到banner里面
           <h1>...</h1>
       </div>
   </div>
   ```

2. ✅ **方案2：修改CSS选择器**
   ```css
   /* 单独选择content，不依赖banner */
   .content {
       position: fixed;
       top: 50%;
       left: 50%;
       transform: translate(-50%, -50%);
       z-index: 1;
   }
   ```

**经验教训**：
- 🔍 **CSS选择器必须与HTML结构匹配**
- 🔍 **使用浏览器开发者工具检查元素是否被选中**
- 🔍 **`.parent .child` = 选中parent内部的child**
- 🔍 **`.parent` + `.child` = 选中兄弟关系**
- 💡 **先设计HTML结构，再写CSS选择器**

**知识点**：
- CSS后代选择器：`.A .B` 选中A内部的B
- CSS子选择器：`.A > .B` 选中A的直接子元素B
- CSS兄弟选择器：`.A + .B` 选中A后面的第一个兄弟B
- 使用开发者工具：Elements → Computed 检查样式是否应用

### 1.2 CSS选择器问题

#### 项目01-问题3：CSS选择器错误（text vs #text）

**现象**：文字样式不生效

**原因**：
- ❌ 使用 `text` 选择器，会选中 `<text>` 标签（SVG）
- ❌ HTML中是 `<h2 id="text">`

**解决方案**：
```css
/* 错误 */
text { color: #fff; }

/* 正确 */
#text { color: #fff; }
```

**知识点**：
- `text` = 标签选择器，选中所有 `<text>` 元素
- `#text` = ID选择器，选中 `id="text"` 的元素
- `.text` = 类选择器，选中 `class="text"` 的元素

## 二、文件和资源管理类

### 2.1 路径问题

#### 项目01-问题1：图片无法显示（Windows反斜杠问题）

**现象**：HTML 中 `<img>` 标签引用的图片不显示

**原因**：
1. ❌ 使用了 Windows 风格的反斜杠 `\`
2. ❌ 图片文件不在项目目录下

**解决方案**：
1. ✅ 将路径中的 `\` 改为 `/`
   ```html
   <!-- 错误 -->
   <img src="assets\issue-1\bg.jpg">

   <!-- 正确 -->
   <img src="assets/issue-1/bg.jpg">
   ```

2. ✅ 将图片文件复制到项目目录
   ```bash
   mkdir -p projects/01-parallax-scrolling/assets/issue-1
   cp assets/issue-1/* projects/01-parallax-scrolling/assets/issue-1/
   ```

**知识点**：
- HTML/CSS/JavaScript 中的路径统一使用正斜杠 `/`
- 反斜杠 `\` 是 Windows 系统路径分隔符，不适用于 Web 开发

---

#### 项目02-问题1：文件名包含空格导致路径问题

**现象**：视频文件无法加载

**原因**：
- ❌ 文件名包含空格：`water stone.mp4`
- ❌ 浏览器可能将空格解析为URL分隔符

**解决方案**：
1. ✅ 重命名文件，去掉空格
   ```bash
   # 重命名文件
   mv "water stone.mp4" "water-stone.mp4"
   ```

2. ✅ 如果必须使用带空格的文件名，使用URL编码
   ```html
   <!-- 使用 %20 代替空格 -->
   <source src="assets/issue-2/water%20stone.mp4">

   <!-- 或者不推荐：整个路径加引号 -->
   <source src="assets/issue-2/water stone.mp4">
   ```

**经验教训**：
- 💡 **Web资源文件名不要使用空格**
- 💡 **使用连字符 `-` 或下划线 `_` 代替空格**
- 💡 **保持文件名简洁，避免特殊字符**

**现象**：

3. ✅ **使用纯深色背景**
   ```css
   body {
       background: #000; /* 纯黑背景，让霓虹效果更明显 */
   }
   ```

**经验教训**：
- 💡 **外部资源（图片、字体）应随项目一起提供**
- 💡 **使用 background-color 作为背景图片的备选方案**
- 💡 **霓虹灯效果需要深色背景才能突出显示**
- 💡 **text-shadow 多层叠加可以创建渐变发光效果**

## 三、设计和布局类

### 3.1 层级问题

#### 项目01-问题4：z-index层级混乱

**现象**：文字被遮罩层覆盖看不见

**原因**：
- ❌ 遮罩层 z-index: 10000
- ❌ 文字 z-index: 1
- ❌ 文字在遮罩层下面

**解决方案**：
```css
/* 正确的层级关系 */
section img      { z-index: 1; }   /* 图片层：最底层 */
#road            { z-index: 2; }   /* 道路：前景 */
#text            { z-index: 3; }   /* 文字：图片之上 */
section::before  { z-index: 10; }  /* 底部遮罩：文字之上 */
section::after   { z-index: 11; }  /* 顶部遮罩：最上层 */
```

**知识点**：
- z-index 值越大，层级越高
- 子元素 z-index 只在同级元素间比较
- 需要考虑父元素的堆叠上下文

### 3.2 响应式设计问题

#### 项目06-问题1：导航栏在小屏幕上无法完整显示

**现象**：
- 5个按钮，每个固定宽度 150px
- 总宽度 750px，在笔记本（1366px）上显示正常
- 但在手机（375px）上会溢出或需要横向滚动

**原因**：
- ❌ 使用固定像素宽度（150px），无法自适应屏幕
- ❌ 没有响应式设计，未针对不同设备调整
- ❌ 视口宽度 < 750px 时会出现内容被裁剪

**解决方案**：
1. ✅ **添加媒体查询**，针对不同屏幕尺寸调整按钮大小
   ```css
   /* 平板横屏（≤ 1199px）*/
   @media (max-width: 1199px) {
       ul li a { width: 120px; height: 70px; font-size: 14px; }
   }

   /* 平板竖屏（≤ 767px）*/
   @media (max-width: 767px) {
       ul li a { width: 100px; height: 60px; font-size: 13px; }
   }

   /* 大屏手机（481-575px）：显示文字*/
   @media (min-width: 481px) and (max-width: 575px) {
       ul li a { width: 80px; height: 50px; font-size: 11px; }
   }

   /* 小屏手机（≤ 480px）：只显示图标*/
   @media (max-width: 480px) {
       ul li a { width: 50px; height: 50px; font-size: 0; }
       ul li a i { font-size: 20px; margin-right: 0; }
   }
   ```

2. ✅ **使用 font-size: 0 隐藏文字**
   - 比使用 `display: none` 或 `visibility: hidden` 更好
   - 保持元素空间，只隐藏文字内容
   - 图标继续显示并保持居中

3. ✅ **渐进式缩小策略**
   - 桌面端：150px × 80px，字体 16px
   - 平板：120px × 70px，字体 14px
   - 小平板：100px × 60px，字体 13px
   - 大屏手机：80px × 50px，字体 11px
   - 小屏手机：50px × 50px，只显示图标

**经验教训**：
- 🔍 **响应式设计是现代 Web 开发的必备技能**
- 🔍 **媒体查询断点应根据实际内容设计**，不要盲目使用常见断点
- 🔍 **隐藏文字的最佳实践**：`font-size: 0` 比 `display: none` 更灵活
- 💡 **从大屏幕开始设计，逐步向下适配**（Mobile First 的反向思路）
- 💡 **在每个断点都要测试悬停动画是否正常**

**知识点**：
- `@media (max-width: Npx)`：最大宽度为 Npx 时应用样式
- `@media (min-width: Npx) and (max-width: Mpx)`：范围断点
- `font-size: 0`：优雅地隐藏文字，保持布局结构
- 响应式设计断点选择：根据内容需求而非设备类型
- 笔记本常见分辨率：1366×768、1920×1080
- 手机常见宽度：375px（iPhone）、360px（Android）

### 3.3 布局问题

#### 项目11-问题2：使用float布局（过时）

**现象**：
- 使用 `float: left` 和 `float: right` 实现左右分栏
- 这是 2000 年代的布局方式

**影响**：
- float 布局已过时，现代项目推荐 Flexbox 或 Grid
- 可能出现高度塌陷问题
- 代码可读性和维护性较差
- 难以实现响应式布局

**为什么使用 Flexbox 更好**：
- ✅ 代码更简洁，无需清除浮动
- ✅ 天然支持响应式设计
- ✅ 对齐和分布更灵活
- ✅ 现代浏览器的性能更好

**推荐改进方案**：
```css
.container {
    display: flex;
    flex-direction: row;
}
```

## 四、最佳实践和代码质量类

### 4.1 代码重复问题

#### 项目13-问题4：加载动画代码重复（技术债）

**现象**：
- 项目05有完整的加载动画（`index.html` + `style.css`）
- 项目13重新创建了 `loading.html` + `loading-style.css`
- 两份代码几乎相同，只是颜色不同
- 违反了DRY（Don't Repeat Yourself）原则

**影响**：
- ❌ 代码重复，维护成本高
- ❌ 如果要修改加载动画效果，需要同时修改多处
- ❌ 未来其他项目也需要加载动画时，会继续复制代码
- ❌ 不符合组件化、模块化开发理念

**差异对比**：
| 特性 | 项目05 | 项目13 |
|------|--------|--------|
| 背景色 | `#2196f3`（蓝色） | `#000`（黑色） |
| 加载条颜色 | `#2196f3`（蓝色） | `#667eea`（紫色） |
| 加载文字 | 无 | "加载中..." |
| 文件大小 | ~5KB | ~4KB |

#### 项目01-问题5：Git推送失败 - 误诊案例

**现象**：`git push` 失败，误以为是没有配置远程仓库链接

**完整过程**：
1. 执行 `git push -u origin main`
2. 提示错误：`Failed to connect to github.com port 443 after 21076 ms: Could not connect to server`
3. 我误以为是因为没有添加远程仓库链接
4. 实际上远程仓库链接早就配置好了
5. 第二次推送成功

**问题分析**：
- ❌ **误判原因**：看到推送失败，第一反应是"是不是没有添加远程链接"
- ✅ **真正原因**：网络不稳定，第一次连接超时
- ✅ **链接状态**：`git remote -v` 显示远程仓库已正确配置

**解决方案**：
1. ✅ **先检查配置，再判断问题**
   ```bash
   # 检查远程仓库是否配置
   git remote -v

   # 如果显示：
   # origin  https://github.com/用户名/仓库名.git (fetch)
   # origin  https://github.com/用户名/仓库名.git (push)
   # 说明配置正确
   ```

2. ✅ **区分两种情况**
   - 情况A：未配置远程仓库
     ```bash
     git remote add origin https://github.com/用户名/仓库名.git
     ```

   - 情况B：网络问题（本次遇到的情况）
     ```bash
     # 稍等片刻后重试
     git push -u origin main
     ```

3. ✅ **网络不稳定时的处理方法**
   - 等待几秒后重试
   - 检查网络连接
   - 尝试使用代理
   - 使用SSH方式替代HTTPS

**经验教训**：
- 🔍 **遇到问题先诊断，再下结论**
- 📊 **使用 `git remote -v` 确认配置状态**
- 🔄 **网络不稳定时，重试往往能解决**
- 💡 **不要被错误信息的表面现象迷惑**

**知识点**：
- `git remote -v`：查看远程仓库配置（重要！）
- `Failed to connect` = 网络问题，不是配置问题
- `fatal: 'origin' does not appear to be a git repository` = 没有配置远程仓库

**6. 视差效果（实时更新）**
```javascript
// 使用GSAP ticker实现每帧更新
tickerFn = function() {
    const strength = activeStrength;  // 0-1的渐变值
    if (strength === 0) return;

    const cursorX = gsap.getProperty(cursor, 'x');
    const cursorY = gsap.getProperty(cursor, 'y');

    corners.forEach((corner, i) => {
        const currentX = gsap.getProperty(corner, 'x');
        const currentY = gsap.getProperty(corner, 'y');

        const targetX = targetCornerPositions[i].x - cursorX;
        const targetY = targetCornerPositions[i].y - cursorY;

        // 渐变跟随（关键算法）
        const finalX = currentX + (targetX - currentX) * strength;
        const finalY = currentY + (targetY - currentY) * strength;

        const duration = strength >= 0.99 ? 0.2 : 0.05;

        gsap.to(corner, {
            x: finalX,
            y: finalY,
            duration: duration,
            ease: 'power1.out',
            overwrite: 'auto'  // 自动覆盖之前的动画
        });
    });
};

gsap.ticker.add(tickerFn);
```

**视差算法**：
- `strength` 从0渐变到1
- `finalX = currentX + (targetX - currentX) * strength`
- 当strength=0时，保持当前位置
- 当strength=1时，到达目标位置
- 中间值实现平滑过渡

**7. 悬停离开处理**

```javascript
    const positions = [
        { x: -cornerSize * 1.5, y: -cornerSize * 1.5 },  // 左上
        { x: cornerSize * 0.5, y: -cornerSize * 1.5 },   // 右上
        { x: cornerSize * 0.5, y: cornerSize * 0.5 },    // 右下
        { x: -cornerSize * 1.5, y: cornerSize * 0.5 }    // 左下
    ];

    const tl = gsap.timeline();
    corners.forEach((corner, index) => {
        tl.to(corner, {
            x: positions[index].x,
            y: positions[index].y,
            duration: 0.3,
            ease: 'power3.out'
        }, 0);  // 0表示所有动画同时开始
    });

    // 延迟后恢复旋转动画
    setTimeout(() => {
        const currentRotation = gsap.getProperty(cursor, 'rotation');
        const normalizedRotation = currentRotation % 360;

        spinTl.kill();
        spinTl = gsap.timeline({ repeat: -1 })
            .to(cursor, {
                rotation: '+=360',
                duration: 2,
                ease: 'none'
            });

        // 先完成当前旋转，再重启循环
        gsap.to(cursor, {
            rotation: normalizedRotation + 360,
            duration: 2 * (1 - normalizedRotation / 360),
            ease: 'none',
            onComplete: () => spinTl.restart()
        });
    }, 50);
};
```

**关键点**：
- 使用`gsap.timeline()`同步四个角的动画
- `}, 0)` 让所有动画同时开始
- 计算当前旋转角度，平滑恢复到循环动画

```javascript
function isMobileDevice() {
```
