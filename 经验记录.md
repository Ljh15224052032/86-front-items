## 项目01：视差滚动效果

### 🐛 问题1：图片无法显示

**现象**：HTML 中 `<img>` 标签引用的图片不显示

**原因**：
1. ❌ 使用了 Windows 风格的反斜杠 `\`
2. ❌ 图片文件不在项目目录下

**解决方案**：
1. ✅ 将路径中的 `\` 改为 `/`
   ```html
   <!-- 错误 -->
   <img src="assets\issue-1\bg.jpg">

   <!-- 正确 -->
   <img src="assets/issue-1/bg.jpg">
   ```

2. ✅ 将图片文件复制到项目目录
   ```bash
   mkdir -p projects/01-parallax-scrolling/assets/issue-1
   cp assets/issue-1/* projects/01-parallax-scrolling/assets/issue-1/
   ```

**知识点**：
- HTML/CSS/JavaScript 中的路径统一使用正斜杠 `/`
- 反斜杠 `\` 是 Windows 系统路径分隔符，不适用于 Web 开发

---

### 🐛 问题2：script标签语法错误

**现象**：JavaScript代码不执行

**原因**：
- ❌ `<script type="text/javascript" let bg=...` 语法错误
- ❌ 在JS中使用了HTML注释 `<!-- -->`

**解决方案**：
1. ✅ 正确的script标签写法
   ```html
   <!-- 引入外部JS文件 -->
   <script src="script.js"></script>

   <!-- 或内联代码 -->
   <script>
       let bg = document.getElementById('bg');
   </script>
   ```

2. ✅ 使用JS注释而非HTML注释
   ```javascript
   // 错误：<!-- 这是HTML注释 -->
   // 正确：// 这是JS单行注释
   /* 正确：这是JS多行注释 */
   ```

**知识点**：
- `<script>` 标签的 `type` 属性后不能直接跟代码
- JS中使用 `//` 或 `/* */` 注释，不能用 `<!-- -->`

### 🐛 问题3：CSS选择器错误

**现象**：文字样式不生效

**原因**：
- ❌ 使用 `text` 选择器，会选中 `<text>` 标签（SVG）
- ❌ HTML中是 `<h2 id="text">`

**解决方案**：
```css
/* 错误 */
text { color: #fff; }

/* 正确 */
#text { color: #fff; }
```

**知识点**：
- `text` = 标签选择器，选中所有 `<text>` 元素
- `#text` = ID选择器，选中 `id="text"` 的元素
- `.text` = 类选择器，选中 `class="text"` 的元素

---

### 🐛 问题4：z-index层级混乱

**现象**：文字被遮罩层覆盖看不见

**原因**：
- ❌ 遮罩层 z-index: 10000
- ❌ 文字 z-index: 1
- ❌ 文字在遮罩层下面

**解决方案**：
```css
/* 正确的层级关系 */
section img      { z-index: 1; }   /* 图片层：最底层 */
#road            { z-index: 2; }   /* 道路：前景 */
#text            { z-index: 3; }   /* 文字：图片之上 */
section::before  { z-index: 10; }  /* 底部遮罩：文字之上 */
section::after   { z-index: 11; }  /* 顶部遮罩：最上层 */
```

**知识点**：
- z-index 值越大，层级越高
- 子元素 z-index 只在同级元素间比较
- 需要考虑父元素的堆叠上下文

### 🐛 问题5：Git推送失败 - 误诊案例

**现象**：`git push` 失败，误以为是没有配置远程仓库链接

**完整过程**：
1. 执行 `git push -u origin main`
2. 提示错误：`Failed to connect to github.com port 443 after 21076 ms: Could not connect to server`
3. 我误以为是因为没有添加远程仓库链接
4. 实际上远程仓库链接早就配置好了
5. 第二次推送成功

**问题分析**：
- ❌ **误判原因**：看到推送失败，第一反应是"是不是没有添加远程链接"
- ✅ **真正原因**：网络不稳定，第一次连接超时
- ✅ **链接状态**：`git remote -v` 显示远程仓库已正确配置

**解决方案**：
1. ✅ **先检查配置，再判断问题**
   ```bash
   # 检查远程仓库是否配置
   git remote -v

   # 如果显示：
   # origin  https://github.com/用户名/仓库名.git (fetch)
   # origin  https://github.com/用户名/仓库名.git (push)
   # 说明配置正确
   ```

2. ✅ **区分两种情况**
   - 情况A：未配置远程仓库
     ```bash
     git remote add origin https://github.com/用户名/仓库名.git
     ```

   - 情况B：网络问题（本次遇到的情况）
     ```bash
     # 稍等片刻后重试
     git push -u origin main
     ```

3. ✅ **网络不稳定时的处理方法**
   - 等待几秒后重试
   - 检查网络连接
   - 尝试使用代理
   - 使用SSH方式替代HTTPS

**经验教训**：
- 🔍 **遇到问题先诊断，再下结论**
- 📊 **使用 `git remote -v` 确认配置状态**
- 🔄 **网络不稳定时，重试往往能解决**
- 💡 **不要被错误信息的表面现象迷惑**

**知识点**：
- `git remote -v`：查看远程仓库配置（重要！）
- `Failed to connect` = 网络问题，不是配置问题
- `fatal: 'origin' does not appear to be a git repository` = 没有配置远程仓库

### 🐛 问题6：Git推送网络问题（补充）

**现象**：`git push` 失败，提示连接超时

**原因**：
- ❌ 网络连接问题
- ❌ 防火墙阻止
- ❌ 需要代理

**解决方案**：
1. ✅ 检查远程仓库配置


3. ✅ 推送到远程仓库
   ```bash
   git push -u origin main
   ```

4. ✅ 如果需要代理


- `git remote add` 添加远程仓库链接
- `git push -u` 设置上游分支并推送
- `-u` 参数首次推送后，以后只需 `git push`

## 项目02：全屏视频背景

### 🐛 问题1：文件名包含空格导致路径问题

**现象**：视频文件无法加载

**原因**：
- ❌ 文件名包含空格：`water stone.mp4`
- ❌ 浏览器可能将空格解析为URL分隔符

**解决方案**：
1. ✅ 重命名文件，去掉空格
   ```bash
   # 重命名文件
   mv "water stone.mp4" "water-stone.mp4"
   ```

2. ✅ 如果必须使用带空格的文件名，使用URL编码
   ```html
   <!-- 使用 %20 代替空格 -->
   <source src="assets/issue-2/water%20stone.mp4">

   <!-- 或者不推荐：整个路径加引号 -->
   <source src="assets/issue-2/water stone.mp4">
   ```

**经验教训**：
- 💡 **Web资源文件名不要使用空格**
- 💡 **使用连字符 `-` 或下划线 `_` 代替空格**
- 💡 **保持文件名简洁，避免特殊字符**

---

### 🐛 问题2：HTML结构与CSS选择器不匹配

**现象**：CSS样式不生效，内容层无法显示

**原因**：
- ❌ CSS使用 `.banner .content` 选择器（查找banner内部的content）
- ❌ HTML中 `.content` 和 `.banner` 是兄弟关系，不是父子关系

**错误代码**：
```html
<!-- HTML结构 -->
<div class="banner">
    <video>...</video>
</div>

<div class="content">  ← 在banner外面！
    <h1>...</h1>
</div>
```

```css
/* CSS选择器 */
.banner .content {  /* 这个选择器找不到元素 */
    z-index: 1;
}
```

**解决方案**：
1. ✅ **方案1：修改HTML结构**（推荐）
   ```html
   <div class="banner">
       <video>...</video>
       <div class="content">  ← 移到banner里面
           <h1>...</h1>
       </div>
   </div>
   ```

2. ✅ **方案2：修改CSS选择器**
   ```css
   /* 单独选择content，不依赖banner */
   .content {
       position: fixed;
       top: 50%;
       left: 50%;
       transform: translate(-50%, -50%);
       z-index: 1;
   }
   ```

**经验教训**：
- 🔍 **CSS选择器必须与HTML结构匹配**
- 🔍 **使用浏览器开发者工具检查元素是否被选中**
- 🔍 **`.parent .child` = 选中parent内部的child**
- 🔍 **`.parent` + `.child` = 选中兄弟关系**
- 💡 **先设计HTML结构，再写CSS选择器**

**知识点**：
- CSS后代选择器：`.A .B` 选中A内部的B
- CSS子选择器：`.A > .B` 选中A的直接子元素B
- CSS兄弟选择器：`.A + .B` 选中A后面的第一个兄弟B
- 使用开发者工具：Elements → Computed 检查样式是否应用

## 项目06：社交媒体悬停导航

### 🐛 问题1：导航栏在小屏幕上无法完整显示

**现象**：
- 5个按钮，每个固定宽度 150px
- 总宽度 750px，在笔记本（1366px）上显示正常
- 但在手机（375px）上会溢出或需要横向滚动

**原因**：
- ❌ 使用固定像素宽度（150px），无法自适应屏幕
- ❌ 没有响应式设计，未针对不同设备调整
- ❌ 视口宽度 < 750px 时会出现内容被裁剪

**解决方案**：
1. ✅ **添加媒体查询**，针对不同屏幕尺寸调整按钮大小
   ```css
   /* 平板横屏（≤ 1199px）*/
   @media (max-width: 1199px) {
       ul li a { width: 120px; height: 70px; font-size: 14px; }
   }

   /* 平板竖屏（≤ 767px）*/
   @media (max-width: 767px) {
       ul li a { width: 100px; height: 60px; font-size: 13px; }
   }

   /* 大屏手机（481-575px）：显示文字*/
   @media (min-width: 481px) and (max-width: 575px) {
       ul li a { width: 80px; height: 50px; font-size: 11px; }
   }

   /* 小屏手机（≤ 480px）：只显示图标*/
   @media (max-width: 480px) {
       ul li a { width: 50px; height: 50px; font-size: 0; }
       ul li a i { font-size: 20px; margin-right: 0; }
   }
   ```

2. ✅ **使用 font-size: 0 隐藏文字**
   - 比使用 `display: none` 或 `visibility: hidden` 更好
   - 保持元素空间，只隐藏文字内容
   - 图标继续显示并保持居中

3. ✅ **渐进式缩小策略**
   - 桌面端：150px × 80px，字体 16px
   - 平板：120px × 70px，字体 14px
   - 小平板：100px × 60px，字体 13px
   - 大屏手机：80px × 50px，字体 11px
   - 小屏手机：50px × 50px，只显示图标

**经验教训**：
- 🔍 **响应式设计是现代 Web 开发的必备技能**
- 🔍 **媒体查询断点应根据实际内容设计**，不要盲目使用常见断点
- 🔍 **隐藏文字的最佳实践**：`font-size: 0` 比 `display: none` 更灵活
- 💡 **从大屏幕开始设计，逐步向下适配**（Mobile First 的反向思路）
- 💡 **在每个断点都要测试悬停动画是否正常**

**知识点**：
- `@media (max-width: Npx)`：最大宽度为 Npx 时应用样式
- `@media (min-width: Npx) and (max-width: Mpx)`：范围断点
- `font-size: 0`：优雅地隐藏文字，保持布局结构
- 响应式设计断点选择：根据内容需求而非设备类型
- 笔记本常见分辨率：1366×768、1920×1080
- 手机常见宽度：375px（iPhone）、360px（Android）

## 项目09：悬停分裂图片

⚠️ **略有瑕疵**：项目基本完成，但存在以下已知问题。

### 问题1：使用外部图片链接

**现象**：图片来自 freepik.com 外部链接

**影响**：
- 可能存在防盗链问题
- 外部链接可能失效
- 加载速度受网络影响

**解决方案**：
1. ✅ 下载图片到本地 `assets` 文件夹
2. ✅ 替换为本地路径：`url('../assets/image.jpg')`
3. ✅ 或使用稳定的 CDN 服务

---

### 问题2：图片尺寸计算复杂

**现象**：
- `background-size: 1200px auto` 需要手动计算
- `background-position: -600px 0` 需要精确计算
- 更换图片时需要重新计算所有参数

**原因**：
- 容器宽度 600px，每半边 300px
- 原始图片宽度 740px
- 需要放大到 1200px 才能完整显示
- 左半边显示 0-600px，右半边显示 600-1200px

**解决方案**：
```css
/* 使用 CSS 变量简化 */
:root {
    --container-width: 600px;
    --original-image-width: 740px;
    --scale-factor: calc(var(--container-width) * 2 / var(--original-image-width));
}

.figure:before,
.figure:after {
    background-size: calc(var(--original-image-width) * var(--scale-factor)) auto;
}
```

---

### 问题3：缺少响应式设计

**现象**：固定宽度 600px，在移动设备上可能溢出

**建议添加**：
```css
@media (max-width: 767px) {
    .box {
        width: 90%;
        height: auto;
        aspect-ratio: 3 / 2;
    }
}
```

---

### 问题4：使用占位文本

**现象**：使用 Lorem ipsum 无意义文本

**建议**：替换为有意义的中文或英文内容，提升用户体验。

---

**经验教训**：
- 💡 **避免使用外部图片链接**，优先使用本地资源
- 💡 **复杂的尺寸计算应使用 CSS 变量或 JavaScript**
- 💡 **响应式设计应从一开始就考虑**
- 💡 **使用真实的示例内容，而非占位符**

**知识点**：
- `rotateX(90deg)`：沿X轴旋转90度

- `background-position` 负值：向左/上移动背景图片
- 伪元素可以分别显示同一背景图的不同部分

## 项目10：霓虹灯文字效果

⚠️ **略有瑕疵**：项目基本完成，但缺少背景图片。

### 问题：缺少背景图片文件

**现象**：
- CSS 中引用了 `url(bg.jpg)` 背景图片
- 项目文件夹中没有提供 `bg.jpg` 文件
- 导致背景显示为空白

**影响**：
- 霓虹灯效果在浅色或白色背景上不够明显
- 视觉效果大打折扣

**解决方案**：
1. ✅ **添加背景图片**
   - 在项目中创建 `assets` 文件夹
   - 添加深色背景图片（如夜景、墙壁等）
   - 更新 CSS 路径：`url('../assets/bg.jpg')`

2. ✅ **设置备选背景色**
   ```css
   body {
       background: url(bg.jpg);
       background-color: #000; /* 如果图片加载失败，显示黑色 */
   }
   ```

3. ✅ **使用纯深色背景**
   ```css
   body {
       background: #000; /* 纯黑背景，让霓虹效果更明显 */
   }
   ```

---

**经验教训**：
- 💡 **外部资源（图片、字体）应随项目一起提供**
- 💡 **使用 background-color 作为背景图片的备选方案**
- 💡 **霓虹灯效果需要深色背景才能突出显示**
- 💡 **text-shadow 多层叠加可以创建渐变发光效果**

**技术亮点**：
- `text-shadow` 多层叠加：10px → 20px → 40px → 80px
- `attr(data-text)` 读取 HTML 属性值作为伪元素内容
- `filter: blur()` 创建模糊光晕效果
- 伪元素配合 z-index 创建多层光晕

**知识点**：
- `attr()` 函数：读取 HTML 元素的属性值
- `filter: blur(Npx)`：高斯模糊滤镜
- `text-shadow`：多层阴影可以叠加，创建渐变效果
- 霓虹灯效果原理：白色文字 + 彩色模糊层 + 背景光晕

### 问题2：使用 float 布局（过时）

**现象**：
- 使用 `float: left` 和 `float: right` 实现左右分栏
- 这是 2000 年代的布局方式

**影响**：
- float 布局已过时，现代项目推荐 Flexbox 或 Grid
- 可能出现高度塌陷问题
- 代码可读性和维护性较差
- 难以实现响应式布局

**为什么使用 Flexbox 更好**：
- ✅ 代码更简洁，无需清除浮动
- ✅ 天然支持响应式设计
- ✅ 对齐和分布更灵活
- ✅ 现代浏览器的性能更好

**推荐改进方案**：
```css
.container {
    display: flex;
    flex-direction: row;
}
```

#### 4. 状态机动画系统
```javascript
let phase = "drawing"; // 当前阶段
let drawProgress = 0;  // 绘制进度
let holdTimer = 0;     // 保持计时器
let fadeAlpha = 1;     // 淡出透明度
let labelAlpha = 0;    // 标签透明度

switch (phase) {
    case "drawing":   // 连线绘制
        drawProgress += 0.004;
        if (drawProgress >= 1.05) phase = "labeling";
        break;
    case "labeling":  // 标签淡入
        labelAlpha = Math.min(1, labelAlpha + 0.02);
        if (labelAlpha >= 1) phase = "holding";
        break;
    case "holding":   // 保持显示
        holdTimer++;
        if (holdTimer > 160) phase = "fading";
        break;
    case "fading":    // 淡出
        fadeAlpha -= 0.012;
        if (fadeAlpha <= 0) {
            currentIndex++; // 切换到下一个星座
            phase = "drawing";
        }
        break;
}
```

**状态转换**：
```
drawing → labeling → holding → fading → drawing (下一个星座)
  4.2秒      0.8秒       2.7秒      1.2秒
```

```javascript
});
```

#### 6. 视差效果（实时更新）
```javascript
// 使用GSAP ticker实现每帧更新
tickerFn = function() {
    const strength = activeStrength;  // 0-1的渐变值
    if (strength === 0) return;

    const cursorX = gsap.getProperty(cursor, 'x');
    const cursorY = gsap.getProperty(cursor, 'y');

    corners.forEach((corner, i) => {
        const currentX = gsap.getProperty(corner, 'x');
        const currentY = gsap.getProperty(corner, 'y');

        const targetX = targetCornerPositions[i].x - cursorX;
        const targetY = targetCornerPositions[i].y - cursorY;

        // 渐变跟随（关键算法）
        const finalX = currentX + (targetX - currentX) * strength;
        const finalY = currentY + (targetY - currentY) * strength;

        const duration = strength >= 0.99 ? 0.2 : 0.05;

        gsap.to(corner, {
            x: finalX,
            y: finalY,
            duration: duration,
            ease: 'power1.out',
            overwrite: 'auto'  // 自动覆盖之前的动画
        });
    });
};

gsap.ticker.add(tickerFn);
```

**视差算法**：
- `strength` 从0渐变到1
- `finalX = currentX + (targetX - currentX) * strength`
- 当strength=0时，保持当前位置
- 当strength=1时，到达目标位置
- 中间值实现平滑过渡

```javascript
    // 移除ticker
    gsap.ticker.remove(tickerFn);

    // 恢复四个角的位置
    const { cornerSize } = constants;
    const positions = [
        { x: -cornerSize * 1.5, y: -cornerSize * 1.5 },  // 左上
        { x: cornerSize * 0.5, y: -cornerSize * 1.5 },   // 右上
        { x: cornerSize * 0.5, y: cornerSize * 0.5 },    // 右下
        { x: -cornerSize * 1.5, y: cornerSize * 0.5 }    // 左下
    ];

    const tl = gsap.timeline();
    corners.forEach((corner, index) => {
        tl.to(corner, {
            x: positions[index].x,
            y: positions[index].y,
            duration: 0.3,
            ease: 'power3.out'
        }, 0);  // 0表示所有动画同时开始
    });

    // 延迟后恢复旋转动画
    setTimeout(() => {
        const currentRotation = gsap.getProperty(cursor, 'rotation');
        const normalizedRotation = currentRotation % 360;

        spinTl.kill();
        spinTl = gsap.timeline({ repeat: -1 })
            .to(cursor, {
                rotation: '+=360',
                duration: 2,
                ease: 'none'
            });

        // 先完成当前旋转，再重启循环
        gsap.to(cursor, {
            rotation: normalizedRotation + 360,
            duration: 2 * (1 - normalizedRotation / 360),
            ease: 'none',
            onComplete: () => spinTl.restart()
        });
    }, 50);
};
```

**关键点**：
- 使用`gsap.timeline()`同步四个角的动画
- `}, 0)` 让所有动画同时开始
- 计算当前旋转角度，平滑恢复到循环动画

#### 9. 移动端检测
```javascript
function isMobileDevice() {
    const hasTouchScreen = 'ontouchstart' in window || navigator.maxTouchPoints > 0;
    const isSmallScreen = window.innerWidth <= 768;
    const userAgent = navigator.userAgent || navigator.vendor || window.opera;
    const mobileRegex = /android|webos|iphone|ipad|ipod|blackberry|iemobile|opera mini/i;
    const isMobileUserAgent = mobileRegex.test(userAgent.toLowerCase());
    return (hasTouchScreen && isSmallScreen) || isMobileUserAgent;
}

}
```

#### 2. gsap.to() - 动画到目标值
```javascript
gsap.to(element, {
```

```javascript
const tl = gsap.timeline({ repeat: -1 })

    .to(element, { scale: 1.5, duration: 0.5 });
```

```javascript
// 添加ticker函数
gsap.ticker.add(function() {

gsap.ticker.remove(fn);
```

```javascript
const x = gsap.getProperty(element, 'x');
```
