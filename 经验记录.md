## 项目01：视差滚动效果

### 🐛 问题1：图片无法显示

**现象**：HTML 中 `<img>` 标签引用的图片不显示

**原因**：
1. ❌ 使用了 Windows 风格的反斜杠 `\`
2. ❌ 图片文件不在项目目录下

**解决方案**：
1. ✅ 将路径中的 `\` 改为 `/`
   ```html
   <!-- 错误 -->
   <img src="assets\issue-1\bg.jpg">

   <!-- 正确 -->
   <img src="assets/issue-1/bg.jpg">
   ```

2. ✅ 将图片文件复制到项目目录
   ```bash
   mkdir -p projects/01-parallax-scrolling/assets/issue-1
   cp assets/issue-1/* projects/01-parallax-scrolling/assets/issue-1/
   ```

**知识点**：
- HTML/CSS/JavaScript 中的路径统一使用正斜杠 `/`
- 反斜杠 `\` 是 Windows 系统路径分隔符，不适用于 Web 开发

### 🐛 问题2：script标签语法错误

**现象**：JavaScript代码不执行

**原因**：
- ❌ `<script type="text/javascript" let bg=...` 语法错误
- ❌ 在JS中使用了HTML注释 `<!-- -->`

**解决方案**：
1. ✅ 正确的script标签写法
   ```html
   <!-- 引入外部JS文件 -->
   <script src="script.js"></script>

   <!-- 或内联代码 -->
   <script>
       let bg = document.getElementById('bg');
   </script>
   ```

2. ✅ 使用JS注释而非HTML注释
   ```javascript
   // 错误：<!-- 这是HTML注释 -->
   // 正确：// 这是JS单行注释
   /* 正确：这是JS多行注释 */
   ```

**知识点**：
- `<script>` 标签的 `type` 属性后不能直接跟代码
- JS中使用 `//` 或 `/* */` 注释，不能用 `<!-- -->`

### 🐛 问题3：CSS选择器错误

**现象**：文字样式不生效

**原因**：
- ❌ 使用 `text` 选择器，会选中 `<text>` 标签（SVG）
- ❌ HTML中是 `<h2 id="text">`

**解决方案**：
```css
/* 错误 */
text { color: #fff; }

/* 正确 */
#text { color: #fff; }
```

**知识点**：
- `text` = 标签选择器，选中所有 `<text>` 元素
- `#text` = ID选择器，选中 `id="text"` 的元素
- `.text` = 类选择器，选中 `class="text"` 的元素

---

### 🐛 问题4：z-index层级混乱

**现象**：文字被遮罩层覆盖看不见

**原因**：
- ❌ 遮罩层 z-index: 10000
- ❌ 文字 z-index: 1
- ❌ 文字在遮罩层下面

**解决方案**：
```css
/* 正确的层级关系 */
section img      { z-index: 1; }   /* 图片层：最底层 */
#road            { z-index: 2; }   /* 道路：前景 */
#text            { z-index: 3; }   /* 文字：图片之上 */
section::before  { z-index: 10; }  /* 底部遮罩：文字之上 */
section::after   { z-index: 11; }  /* 顶部遮罩：最上层 */
```

**知识点**：
- z-index 值越大，层级越高
- 子元素 z-index 只在同级元素间比较
- 需要考虑父元素的堆叠上下文

### 🐛 问题5：Git推送失败 - 误诊案例

**现象**：`git push` 失败，误以为是没有配置远程仓库链接

**完整过程**：
1. 执行 `git push -u origin main`
2. 提示错误：`Failed to connect to github.com port 443 after 21076 ms: Could not connect to server`
3. 我误以为是因为没有添加远程仓库链接
4. 实际上远程仓库链接早就配置好了
5. 第二次推送成功

**问题分析**：
- ❌ **误判原因**：看到推送失败，第一反应是"是不是没有添加远程链接"
- ✅ **真正原因**：网络不稳定，第一次连接超时
- ✅ **链接状态**：`git remote -v` 显示远程仓库已正确配置

**解决方案**：
1. ✅ **先检查配置，再判断问题**
   ```bash
   # 检查远程仓库是否配置
   git remote -v

   # 如果显示：
   # origin  https://github.com/用户名/仓库名.git (fetch)
   # origin  https://github.com/用户名/仓库名.git (push)
   # 说明配置正确
   ```

2. ✅ **区分两种情况**
   - 情况A：未配置远程仓库
     ```bash
     git remote add origin https://github.com/用户名/仓库名.git
     ```

   - 情况B：网络问题（本次遇到的情况）
     ```bash
     # 稍等片刻后重试
     git push -u origin main
     ```

3. ✅ **网络不稳定时的处理方法**
   - 等待几秒后重试
   - 检查网络连接
   - 尝试使用代理
   - 使用SSH方式替代HTTPS

**经验教训**：
- 🔍 **遇到问题先诊断，再下结论**
- 📊 **使用 `git remote -v` 确认配置状态**
- 🔄 **网络不稳定时，重试往往能解决**
- 💡 **不要被错误信息的表面现象迷惑**

**知识点**：
- `git remote -v`：查看远程仓库配置（重要！）
- `Failed to connect` = 网络问题，不是配置问题
- `fatal: 'origin' does not appear to be a git repository` = 没有配置远程仓库

### 🐛 问题6：Git推送网络问题（补充）

**现象**：`git push` 失败，提示连接超时

**原因**：
- ❌ 网络连接问题
- ❌ 防火墙阻止
- ❌ 需要代理

**解决方案**：
1. ✅ 检查远程仓库配置
   ```bash
   git remote -v
   ```

2. ✅ 确认远程仓库已添加
   ```bash
   git remote add origin https://github.com/用户名/仓库名.git
   ```

3. ✅ 推送到远程仓库
   ```bash
   git push -u origin main
   ```

4. ✅ 如果需要代理
   ```bash
   git config --global http.proxy http://代理地址:端口
   git config --global https.proxy https://代理地址:端口
   ```

**知识点**：
- `git remote add` 添加远程仓库链接
- `git push -u` 设置上游分支并推送
- `-u` 参数首次推送后，以后只需 `git push`

## 项目02：全屏视频背景

### 🐛 问题1：文件名包含空格导致路径问题

**现象**：视频文件无法加载

**原因**：
- ❌ 文件名包含空格：`water stone.mp4`
- ❌ 浏览器可能将空格解析为URL分隔符

**解决方案**：
1. ✅ 重命名文件，去掉空格
   ```bash
   # 重命名文件
   mv "water stone.mp4" "water-stone.mp4"
   ```

2. ✅ 如果必须使用带空格的文件名，使用URL编码
   ```html
   <!-- 使用 %20 代替空格 -->
   <source src="assets/issue-2/water%20stone.mp4">

   <!-- 或者不推荐：整个路径加引号 -->
   <source src="assets/issue-2/water stone.mp4">
   ```

**经验教训**：
- 💡 **Web资源文件名不要使用空格**
- 💡 **使用连字符 `-` 或下划线 `_` 代替空格**
- 💡 **保持文件名简洁，避免特殊字符**

---

### 🐛 问题2：HTML结构与CSS选择器不匹配

**现象**：CSS样式不生效，内容层无法显示

**原因**：
- ❌ CSS使用 `.banner .content` 选择器（查找banner内部的content）
- ❌ HTML中 `.content` 和 `.banner` 是兄弟关系，不是父子关系

**错误代码**：
```html
<!-- HTML结构 -->
<div class="banner">
    <video>...</video>
</div>

<div class="content">  ← 在banner外面！
    <h1>...</h1>
</div>
```

```css
/* CSS选择器 */
.banner .content {  /* 这个选择器找不到元素 */
    z-index: 1;
}
```

**解决方案**：
1. ✅ **方案1：修改HTML结构**（推荐）
   ```html
   <div class="banner">
       <video>...</video>
       <div class="content">  ← 移到banner里面
           <h1>...</h1>
       </div>
   </div>
   ```

2. ✅ **方案2：修改CSS选择器**
   ```css
   /* 单独选择content，不依赖banner */
   .content {
       position: fixed;
       top: 50%;
       left: 50%;
       transform: translate(-50%, -50%);
       z-index: 1;
   }
   ```

**经验教训**：
- 🔍 **CSS选择器必须与HTML结构匹配**
- 🔍 **使用浏览器开发者工具检查元素是否被选中**
- 🔍 **`.parent .child` = 选中parent内部的child**
- 🔍 **`.parent` + `.child` = 选中兄弟关系**
- 💡 **先设计HTML结构，再写CSS选择器**

**知识点**：
- CSS后代选择器：`.A .B` 选中A内部的B
- CSS子选择器：`.A > .B` 选中A的直接子元素B
- CSS兄弟选择器：`.A + .B` 选中A后面的第一个兄弟B
- 使用开发者工具：Elements → Computed 检查样式是否应用
